# Shepardizing Workflow

Use this for treatment review of cases, statutes, and rules.

## Core Principle

Shepardize by proposition when possible. A case can have negative treatment on one point and remain valid for another. Do not report treatment mechanically without checking whether it affects the proposition being cited.

## Case Treatment

1. Open the case in Lexis.
2. Review the Shepard's signal and reason for signal.
3. Open the full Shepard's report.
4. Review:
   - appellate history
   - citing decisions
   - negative treatment
   - analysis phrases
   - jurisdiction filters
   - publication status
   - timeline
   - depth of discussion
5. If a relevant headnote exists, use Shepardize - Narrow by this Headnote.
6. Search within citing decisions for the proposition terms.
7. Determine whether any negative treatment affects the exact sentence in the draft.

If browser automation cannot reliably read the case or Shepard's page:

1. Reopen the case by exact citation.
2. Try the Shepardize path from the opened document or a `shep:` citation search.
3. Attempt a narrow treatment, headnote, or citing-decision view before full-page extraction.
4. If the page still times out, use Lexis delivery, export, download, or print for the Shepard's report or narrow treatment view.
5. Parse the exported report locally for the signal, citing decision, proposition affected, and recommendation.
6. Record export/download failures only in the working treatment notes or search log.

## Statute / Rule Treatment

1. Open the current statute or rule.
2. Confirm current text and effective date.
3. Review citing decisions and any legislative or rule history available.
4. Search within citing decisions for the procedural use or disputed issue.
5. Confirm no binding authority limits the use being made in the draft.

## Treatment Categories

Report treatment in plain English:

- no negative treatment found affecting this proposition
- negative treatment exists but appears unrelated to this proposition
- cautionary treatment exists and should be reviewed before relying heavily
- authority has been limited or distinguished on this proposition
- authority is unsafe for this proposition

## Output

Use this format:

| Authority | Shepard's Signal / Treatment | Proposition Checked | Effect On Proposition | Recommendation |
|---|---|---|---|---|

Keep treatment/status notes in a Shepard/treatment working memo unless the user asks to include them in the authority bank or final memo. Do not place timeout or export-limitation caveats in final attorney work product unless the user expressly requests them.

## Research Expansion

Use Shepard's to find more authority:

- citing decisions by same jurisdiction
- citing decisions that discuss the same headnote
- deeply discussed citing decisions
- newer cases using the same proposition
- secondary sources citing the authority

## Caution

If the Shepard's result is complex, do not collapse it into a single label. Explain the actual treatment and why it does or does not matter.
